# Profile 结构字段描述

本文档定义了用于角色配置的 `profile` JSON 文件的结构、字段函数及其建议的属性值。

## 1. `identity_layer` (身份信息层)

* **函数**: 定义角色的基础背景和身份。

### `name`
* **函数**: 角色的唯一中文姓名。
* **属性值**: 字符串 (String)。

### `occupation`
* **函数**: 角色的职业身份，直接影响其沟通内容和知识背景。
* **属性值**: 字符串 (String)，例如: "资深后端工程师", "产品经理", "SRE", "研发总监"。

---

## 2. `behavioral_patterns` (行为模式层)

* **函数**: 描述角色的行为倾向，重点在于沟通方式。

### `communication_style`
* **函数**: 包含一系列用于定义角色沟通风格的子属性。
* **子字段详情**:

| 字段 | 函数 | 属性值 (枚举) |
| :--- | :--- | :--- |
| `formality` (正式程度) | 描述沟通风格的严肃性、结构性以及与他人的社交距离感。 | `高`, `中`, `低` |
| `verbosity` (话量程度) | 描述角色输出信息的详细度或长度，即“话多”还是“话少”。 | `高`, `中`, `低` |
| `humor` (幽默感) | 描述角色在沟通中是否倾向于使用幽默、玩笑或俏皮话。 | `高`, `中`, `低`, `无` |
| `jargon_usage` (专业术语使用) | 描述角色使用专业术语、行业“黑话”或技术行话的频率。 | `高`, `中`, `低` |
| `casual_language_usage` (口语/俚语使用) | 描述角色使用日常口语、网络俚语或非正式词汇（如 "yyds", "gg"）的频率。 | `高`, `中`, `低` |

---

## 3. `knowledge_and_beliefs` (知识信念层)

* **函数**: 定义角色的知识体系，主要体现为其专业技能。

### `domain_knowledge`
* **函数**: 一个列表，存储该角色掌握的核心技能、工具或专业知识点（硬技能）。
* **属性值**: 字符串数组 (Array of Strings)，例如: `["Java", "分布式系统", "MySQL"]` 或 `["用户研究", "PRD", "敏捷开发"]`。
